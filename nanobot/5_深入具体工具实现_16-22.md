## 16. 文件工具 (`agent/tools/filesystem.py`)
### 16.1 共享基类 `_FsTool` + 路径安全
所有文件工具继承自 `_FsTool`，共享路径解析逻辑：
```python
class _FsTool(Tool):
    def __init__(self, workspace, allowed_dir, extra_allowed_dirs):
        self._workspace = workspace
        self._allowed_dir = allowed_dir        # 限制访问的根目录
        self._extra_allowed_dirs = extra_allowed_dirs  # 额外允许的目录（如 builtin skills）
    def _resolve(self, path: str) -> Path:
        return _resolve_path(path, self._workspace, self._allowed_dir, self._extra_allowed_dirs)
```
**`_resolve_path()` 的路径解析流程**：
```python
def _resolve_path(path, workspace, allowed_dir, extra_allowed_dirs):
    p = Path(path).expanduser()          # 1. 展开 ~
    if not p.is_absolute() and workspace:
        p = workspace / p                 # 2. 相对路径 → 基于 workspace 解析
    resolved = p.resolve()               # 3. 解析符号链接，得到绝对路径
    if allowed_dir:
        # 4. 安全检查：必须在允许的目录下
        if not any(_is_under(resolved, d) for d in all_dirs):
            raise PermissionError(...)
    return resolved
```
**`_is_under()` 用 `relative_to()` 检查**：
```python
def _is_under(path, directory):
    try:
        path.relative_to(directory.resolve())  # 能算出相对路径 → 在目录下
        return True
    except ValueError:
        return False  # 抛异常 → 不在目录下
```
**安全模型**：当 `restrict_to_workspace=True` 时，`allowed_dir` 设为 workspace，所有文件操作都被限制在 workspace 内。`extra_allowed_dirs` 用于允许读取 builtin skills 目录（ReadFileTool 专有）。
### 16.2 `ReadFileTool` — 读文件

```python
    _DEFAULT_LIMIT = 2000  # 默认最大行数
```
**分页机制**：
```python
async def execute(self, path, offset=1, limit=None):
    all_lines = fp.read_text().splitlines()
    start = offset - 1  # 1-indexed → 0-indexed
    end = min(start + (limit or 2000), total)
    numbered = [f"{start + i + 1}| {line}" for i, line in enumerate(all_lines[start:end])]
```
输出格式：`1| 第一行内容`，带行号，方便 LLM 引用具体行。
**字符数截断**：如果结果超过 128K 字符，逐行截断到限制内，并提示 `(Showing lines X-Y of Z. Use offset=Y+1 to continue.)`。

### 16.4 `EditFileTool` — 编辑文件（最复杂的文件工具）
**核心难点：模糊匹配**
LLM 提供的 `old_text` 可能和文件中的实际内容有细微差异（缩进、换行符等）。`_find_match()` 实现了两级匹配：
```python
def _find_match(content, old_text):
    # 第一级：精确匹配
    if old_text in content:
        return old_text, content.count(old_text)
    # 第二级：忽略缩进的滑动窗口匹配
    stripped_old = [l.strip() for l in old_text.splitlines()]
    for i in range(len(content_lines) - len(stripped_old) + 1):
        window = content_lines[i : i + len(stripped_old)]
        if [l.strip() for l in window] == stripped_old:
            candidates.append("\n".join(window))
```
**换行符处理**：
```python
raw = fp.read_bytes()
uses_crlf = b"\r\n" in raw                    # 检测是否 Windows 换行
content = raw.decode("utf-8").replace("\r\n", "\n")  # 统一为 LF
# ... 匹配和替换 ...
if uses_crlf:
    new_content = new_content.replace("\n", "\r\n")  # 还原为 CRLF
```
**多重匹配保护**：
```python
if count > 1 and not replace_all:
    return f"Warning: old_text appears {count} times. Provide more context or set replace_all=true."
```
如果 `old_text` 匹配到多处但没设 `replace_all=True`，拒绝执行，防止误改。
**找不到时的智能提示**：
```python
def _not_found_msg(old_text, content, path):
    # 用 difflib 找最相似的片段
    best_ratio, best_start = 0.0, 0
    for i in range(...):
        ratio = difflib.SequenceMatcher(None, old_lines, lines[i:i+window]).ratio()
        if ratio > best_ratio:
            best_ratio, best_start = ratio, i
    if best_ratio > 0.5:
        # 返回 unified diff，帮助 LLM 理解差异
        diff = difflib.unified_diff(...)
        return f"Error: old_text not found. Best match ({best_ratio:.0%}) at line {best_start+1}:\n{diff}"
```
这非常实用——LLM 看到 diff 后可以修正 `old_text` 重试。

## 17. Shell 工具 (`agent/tools/shell.py`)
### 17.1 安全防护体系（三层）
**第一层：危险命令黑名单**
```python
deny_patterns = [
    r"\brm\s+-[rf]{1,2}\b",          # rm -rf
    r"\bdd\s+if=",                   # dd if=
    r">\s*/dev/sd",                  # 写磁盘
    r"\b(shutdown|reboot|poweroff)\b",  # 关机/重启
    r":\(\)\s*\{.*\};\s*:",          # fork bomb
]
```
用正则匹配，覆盖了最常见的破坏性命令。
**第二层：SSRF 防护**
```python
from nanobot.security.network import contains_internal_url
if contains_internal_url(cmd):
    return "Error: internal/private URL detected"
```
防止通过 `curl http://169.254.169.254/` 等方式访问内网资源。
**第三层：工作目录限制**
```python
if self.restrict_to_workspace:
    if "..\\" in cmd or "../" in cmd:
        return "Error: path traversal detected"
    # 提取命令中的绝对路径，检查是否在工作目录内
    for raw in self._extract_absolute_paths(cmd):
        p = Path(raw).expanduser().resolve()
        if p.is_absolute() and cwd_path not in p.parents and p != cwd_path:
            return "Error: path outside working dir"
```
`_extract_absolute_paths()` 用正则提取命令中的绝对路径（支持 POSIX `/path`、Windows `C:\path`、`~` 三种格式）。
### 17.2 命令执行
```python
process = await asyncio.create_subprocess_shell(
    command,
    stdout=asyncio.subprocess.PIPE,
    stderr=asyncio.subprocess.PIPE,
    cwd=cwd,
    env=env,
)
stdout, stderr = await asyncio.wait_for(
    process.communicate(),
    timeout=effective_timeout,
)
```
用 `asyncio.create_subprocess_shell` 创建子进程（支持 shell 语法如管道 `|`），`wait_for` 实现超时。
### 17.3 超时处理
```python
except asyncio.TimeoutError:
    process.kill()                    # 先 kill
    await asyncio.wait_for(process.wait(), timeout=5.0)  # 等进程退出（最多 5 秒）
    return f"Error: Command timed out after {effective_timeout} seconds"
```
kill 后再 wait，避免僵尸进程。
### 17.4 输出截断（头+尾）
```python
if len(result) > max_len:
    half = max_len // 2
    result = result[:half] + f"\n\n... ({len(result) - max_len:,} chars truncated) ...\n\n" + result[-half:]
```
保留输出的开头和结尾，中间截断。这样既能看到命令的初始输出（通常包含错误信息），又能看到最终结果。
### 17.5 `path_append` — PATH 扩展
```python
if self.path_append:
    env["PATH"] = env.get("PATH", "") + os.pathsep + self.path_append
```
允许在 PATH 中追加额外路径，用于访问特定工具（如 Homebrew 安装的命令）。

## 21. Cron 工具 (`agent/tools/cron.py`)
### 21.1 三种操作
```python
async def execute(self, action, message, every_seconds, cron_expr, tz, at, job_id):
    if action == "add":    return self._add_job(...)
    elif action == "list":  return self._list_jobs()
    elif action == "remove": return self._remove_job(job_id)
```
### 21.2 `_add_job()` — 创建定时任务
```python
def _add_job(self, message, every_seconds, cron_expr, tz, at):
    # 构建调度
    if every_seconds:
        schedule = CronSchedule(kind="every", every_ms=every_seconds * 1000)
    elif cron_expr:
        schedule = CronSchedule(kind="cron", expr=cron_expr, tz=tz)
    elif at:
        dt = datetime.fromisoformat(at)
        schedule = CronSchedule(kind="at", at_ms=int(dt.timestamp() * 1000))
        delete_after = True  # 一次性任务，执行后删除
    job = self._cron.add_job(
        name=message[:30],       # 任务名取消息前 30 字符
        schedule=schedule,
        message=message,
        deliver=True,            # 需要投递给用户
        channel=self._channel,   # 投递到当前会话
        to=self._chat_id,
        delete_after_run=delete_after,
    )
```
**`deliver=True`**：表示任务触发时需要通过 agent 循环处理（而不是静默执行）。`channel` 和 `to` 确保结果发送到创建任务时的会话。
### 21.3 防止递归创建
```python
self._in_cron_context: ContextVar[bool] = ContextVar("cron_in_context", default=False)
async def execute(self, action, ...):
    if action == "add":
        if self._in_cron_context.get():
            return "Error: cannot schedule new jobs from within a cron job execution"
```
用 `ContextVar` 标记当前是否在 cron 回调中执行。如果在 cron 任务执行过程中 agent 试图创建新任务，会被拒绝，防止无限递归。

## 22. MCP 工具 (`agent/tools/mcp.py`)
### 22.1 MCP 是什么
MCP（Model Context Protocol）是一个开放标准，允许 AI agent 连接外部工具服务器。比如可以连接 GitHub MCP Server、数据库 MCP Server 等，获得额外的工具能力。
### 22.2 `MCPToolWrapper` — 适配器模式
```python
class MCPToolWrapper(Tool):
    def __init__(self, session, server_name, tool_def, tool_timeout=30):
        self._session = session          # MCP 客户端会话
        self._original_name = tool_def.name
        self._name = f"mcp_{server_name}_{tool_def.name}"  # 加前缀避免命名冲突
        self._description = tool_def.description
        self._parameters = tool_def.inputSchema  # 直接用 MCP 的 JSON Schema
    async def execute(self, **kwargs):
        result = await asyncio.wait_for(
            self._session.call_tool(self._original_name, arguments=kwargs),
            timeout=self._tool_timeout,
        )
        # 提取文本内容
        parts = []
        for block in result.content:
            if isinstance(block, types.TextContent):
                parts.append(block.text)
        return "\n".join(parts)
```
MCP 工具被包装成 nanobot 的 Tool 子类，注册到 ToolRegistry 中。对 LLM 来说，MCP 工具和内置工具没有区别——都是通过 `tool_calls` 调用。
### 22.3 `connect_mcp_servers()` — 连接管理
```python
async def connect_mcp_servers(mcp_servers, registry, stack):
    for name, cfg in mcp_servers.items():
        # 自动检测传输类型
        if cfg.command:
            transport_type = "stdio"       # 本地命令
        elif cfg.url.endswith("/sse"):
            transport_type = "sse"          # Server-Sent Events
        else:
            transport_type = "streamableHttp"  # HTTP 流式
        # 创建传输层
        if transport_type == "stdio":
            read, write = await stdio_client(params)
        elif transport_type == "sse":
            read, write = await sse_client(url)
        elif transport_type == "streamableHttp":
            read, write = await streamable_http_client(url)
        # 创建 MCP 会话
        session = await ClientSession(read, write)
        await session.initialize()
        # 注册工具
        tools = await session.list_tools()
        for tool_def in tools.tools:
            wrapper = MCPToolWrapper(session, name, tool_def)
            registry.register(wrapper)
```
支持三种传输协议：
- **stdio**：通过本地命令启动 MCP 服务器（如 `npx @modelcontextprotocol/server-github`）
- **sse**：通过 Server-Sent Events 连接远程 MCP 服务器
- **streamableHttp**：通过 HTTP 流式连接
### 22.4 工具过滤
```python
enabled_tools = set(cfg.enabled_tools)
allow_all_tools = "*" in enabled_tools
for tool_def in tools.tools:
    if not allow_all_tools and tool_def.name not in enabled_tools:
        continue  # 跳过未启用的工具
    registry.register(wrapper)
```
配置中可以指定 `enabledTools` 来选择性地注册 MCP 工具，而不是全部注册。
### 22.5 CancelledError 处理
```python
except asyncio.CancelledError:
    task = asyncio.current_task()
    if task is not None and task.cancelling() > 0:
        raise  # 真正的外部取消（/stop）→ 传播
    logger.warning("MCP tool was cancelled by server/SDK")
    return "(MCP tool call was cancelled)"  # MCP SDK 内部取消 → 吞掉
```
MCP SDK 的 anyio 取消作用域可能导致意外的 `CancelledError`。这里区分了两种情况：用户主动 `/stop` 的取消会传播，MCP SDK 内部的取消会被吞掉并返回提示。
